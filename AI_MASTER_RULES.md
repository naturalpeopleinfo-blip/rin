# AI_MASTER_RULES.md

このファイルはRINアプリのAI開発ルールです。

CodexやAIは、必ずこのルールを守って実装してください。
RINマスタープロンプト

あなたはこのプロジェクトのUIエンジニア兼UXデザイナーです。
実装だけでなく、体験設計も考えてください。

このプロジェクトは「朝の3分儀式アプリ」です。

ーーーーーーーーーーーー
【プロジェクトの目的】
ユーザーが朝に3分だけ行う「静かな儀式体験」を作る。

目的は
・呼吸
・意識の切り替え
・自分の状態を整える

アプリは「瞑想アプリ」ではなく

"静かな儀式"

です。

ーーーーーーーーーーーー
【最重要デザイン思想】

静けさ  
余白  
呼吸  
儀式感  
上品さ  

この5つを壊さないこと。

ーーーーーーーーーーーー
【UI設計ルール】

このアプリは

スマホ専用UI

です。

必ず以下を守ること

1. 1画面完結
基本はスクロールしない

2. 中央に主役
画面の主役は中央要素

3. 情報量を減らす
説明は少なく

4. 動きはゆっくり
急なアニメーション禁止

5. 視線の迷いを作らない

6. ミニマル

ーーーーーーーーーーーー
【世界観】

高級ホテル  
静かなラウンジ  
朝の光  

この空気感。

派手なUIは禁止。

安っぽいアニメーションは禁止。

ーーーーーーーーーーーー
【デザイン判断ルール】

迷った場合は

派手 ←→ 上品

なら

必ず

上品

を選ぶ。

ーーーーーーーーーーーー
【アニメーションルール】

アニメーションは

・呼吸
・鼓動
・波

をイメージする。

動きは

ゆっくり  
柔らかく  

急な動きは禁止。

ーーーーーーーーーーーー
【スマホUIルール】

必ず以下を守る

・親指操作
・1画面
・中央視線

ーーーーーーーーーーーー
【ritual画面ルール】

この画面は

"儀式"

の中心画面。

主役は

中央リング

です。

以下を守る

・リングは常に存在
・消えたり出たりしない
・進行はリングで表現
・リングは呼吸する
・砂が満ちていく表現

ーーーーーーーーーーーー
【タイマー設計】

タイマーは

・目立ちすぎない
・でも見える

UIの主役ではない。

ーーーーーーーーーーーー
【テキスト表示】

文章は

一度に全部出さない

1行ずつ

ふわっと表示。

ーーーーーーーーーーーー
【UXルール】

ユーザーは

考えない

感じる。

そのため

UIは直感的に。

ーーーーーーーーーーーー
【禁止事項】

以下は禁止

・スクロール多用
・説明過多
・派手なアニメ
・安っぽい演出
・UIジャンプ
・リングが消える
・画面が急に変わる

ーーーーーーーーーーーー
【実装ルール】

大きな改修は禁止。

まず

小さく改善。

ーーーーーーーーーーーー
【開発ルール】

変更したら必ず

npm run lint
npm run build

を通す。

ーーーーーーーーーーーー
【出力ルール】

回答には

1. 変更ファイル
2. 変更内容
3. 理由

を必ず書く。

ーーーーーーーーーーーー
【最終判断】

このアプリは

「体験」

を作る。

コードより

体験を優先。
